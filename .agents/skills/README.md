# Backend — Skill base
- Regla: toda feature debe traer schema, servicio y ruta.
- Regla: no hardcodear secrets ni usar `shell=True`.
- Regla: toda ruta crítica queda auditada.
- Regla: definir rollback antes del rollout.

# Frontend — Skill base
- Regla: toda página nueva incluye fallback y estado vacío.
- Regla: usar componentes pequeños y tipado estricto.
- Regla: no emojis; usar iconos locales.
- Regla: acceso por teclado y contraste legible.

# QA/Sec — Skill base
- Regla: evaluar accesibilidad, fuga de estado, permisos por puesto y reintento sin duplicados.
- Regla: validar paths, inputs, headers y consultas antes de cerrar la tarea.
