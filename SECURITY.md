# Security Policy

## Reportar vulnerabilidades
- Abrir issue privado en GitHub con detalles técnicos
- Incluir pasos de reproducción y evidencia
- No exponer secrets en la issue

## Parcheo
- Releases de seguridad por tag `vSEC-*`
- Rotación de `SECRET_KEY` por despliegue
- SLA: 24 horas para respuesta inicial
